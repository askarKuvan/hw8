About this app

Instructions

I have implemented several versions of control for this app. Keys for each version don't overlap, but in order to switch to other version refresh of the page is needed.

  • 5-keys: Can be done using 4 arrow buttons and any of two Enter buttons for clickSelectedItem
  • 3-keys: Can be done using left(selectNext) and right(selectPrevious) and any of two Enter buttons for clickSelectedItem
  • 2-keys: Can be done using left(selectNext) and any of two Enter buttons for clickSelectedItem
  • 2-keys alternative: In this version, there are two ways that space button works depending on what is selected. At first, by pressing space button you can switch between rows.

    After you switch to row you need, you can press 'alt' button to select the row. Now you can switch between buttons inside that row using space button and select the button using 'alt' button.

    If you want to change from button selection back to row selection, then you need to quickly press space button twice.

  • 1-key: After pressing '0' button, app will start switching between buttons itself. If you want to clickSelectedItem while it's "selected" (yellow), then you need press '0' button.

Test Equations

Following equations were used in the test to use all buttons of app. As you might notice, after each equation I made it required to press '=' button and then 'C' button.

  • 25*34 = C
  • 60+9 = C
  • 71-8 = C
  • 128/4 = C
  • 8*3/2+10-12 = C

Test Equations

I tested each version 5 times for all five equations. Two different ways of input was tested: 1) with any fingers of one hand, 2) with any part of closed fist of one hand.

I measured time using javascript: by pressing 's' button at the beginning of experiment and pressing it again at the end. After pressing 's' for the second time, alert that shows how many milliseconds have passed occurs.

Observations, Comparisons and Conclusion

  • 5-keys: Using fist took much more time because four arrow buttons were close to each other and using fist uncomfortable.
  • 3-keys: Finger and fist inputs seem to have similar time results. Maybe 'finger-input' version should have taken less time, but sometimes my brain was mistakenly pressing disabled up and down arrow buttons from previous results.
  • 2-keys: In this version there is no selectPrevious button. As the result, it took a lot of time to press C button each time after pressing = button. Else, it's really uncomfortable if you pass over the button you wanted to press trying to speed up the selection.
  • 2-keys alternative: This version was supposed to reduce time comparing to 2-keys: version, because it's easier to navigate. However, getting used to different type of navigation was time-consuming. Else, because double press was used for space button, sometimes app was confusing two consequent single presses with double press.
  • 1-key (750ms interval): Here you have to wait until app selects the button, cause you can't select it yourself. Because of it you feel luck of control, especially when you miss your opportunity to press the button and you have to wait again. Else, it quickly gets boring and tiring for eyes while waiting for selection.
  • 1-key (500ms interval): Although it may seem like that decreasing selection interval should decrease task time, it doesn't work that way. It's because with faster selection number of mistakes increases.

Finally, last thing to mention is that time using fist-input was always higher than time using finger-input, which is expected behaviour.
